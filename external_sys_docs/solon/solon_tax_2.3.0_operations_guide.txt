14. Appendix A 
14.1. Amplio Administration 
This section describes the administration user-interfaces for the Amplio Applications used within SOLON TAX ©. 
The user-interface is a shared component used for administrating both the Amplio Process Engine and Revenue 
Management Batch. 

For the Amplio Process Engine, it is responsible for administrating processes and events. The user-interfaces can be 
used for both getting an overview of what is executed across all engines but are primarily used for handling errors, 
deploying process definitions, configuration of the engine, or suspending, cancelling, or restarting processes. 

For the Revenue Management Batch, it is responsible for administrating batch jobs, including scheduling, monitoring, 
and error handling.

14.1.5.2. Active and Inactive System Parameter Instances 
A system parameter instance can either be active or inactive based on its validity period. An instance is active, if the 
reporting date falls within the instance’s validity period. If the reporting date is not within the instance’s validity period, 
the instance will be inactive, and it will not be possible to report or access the instance outside a historical overview. 
Other instances can replace the previous one after it expires, allowing reporting to continue. For example, when 
reporting a code across different instances with different validity periods. 

 
 
An inactive instance is not a deleted instance, as it will still be visible to the administrator in the 

history. 

An active instance will always have a current validity period, and it is possible to edit the validity period of an inactive 
instance to make it active. Overlapping validity periods are handled based on the principles described in section 
Overlap with new instances. 

14.1.5.3. Technical History 
All system parameters have a technical history, so it is always possible to see previous versions of the configuration 
and which user made which changes and when. This includes amongst other things, the creation time, the 
modification time (when an instance was edited), and the publication time (when the instance was deployed to the 
production environment).
