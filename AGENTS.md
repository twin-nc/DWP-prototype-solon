## Agents

Agents are role definitions that govern how Claude approaches a category of work.
Codex should use them as project role guidance: when a task matches an agent description, or the user names an agent, read the authoritative definition in `.claude/agents/<agent>.md` and apply its mission, scope, guardrails, escalation rules, and output contract for that work.
Codex native sub-agent types are separate from these project roles; these definitions still govern how Codex should reason, implement, review, and report within this repository.
